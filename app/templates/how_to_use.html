How to use the Network Configuration Generator

The primary function of this Web service is to create many configurations based on a common template file. It's like a “next generation” mass mailing function for configuration files.

Verify the overall status of the appliance

To verify the overall status of the appliance, please take a look at the Appliance Status page. It should look similar to the following screenshot.

Please note: If the redis server or the local celery worker is not available, the export functions will not work. Furthermore, this view works only correct, if the web service is deployed using one of the setup scripts.

1. Create a new Project

The entire data structure is build around Projects. You can create a new Project on the homepage or in the sidebar of the application. The dialog to create a new Project is straightforward.

2. Create one or more Config Templates

After you've created a Project, you can add one or more Config Templates. You define a configuration template using the Template Syntax used within this Web service. Variables (${ some_value }) are dynamically detected and created for the use within the configuration template.

3. Add as many Template Value Sets as you need

The overview about a Config Template contains a list with all template variables, the associated Template Value Sets and the original configuration template.

By default, a hostname variable is always created. This variable equals the name of the Template Value Set and must be unique within a single Config Template.

You have two options to create/edit a Template Value Set as seen in the table.

3.1 Add a single Template Value Set

If you click "Create the first Template Value Set", you can create a single set as the seen in the following dialog.

3.2 Add multiple Template Values Sets using CSV

To add or edit multiple Template Value Sets, you can use the "add/edit Template Value Sets (CSV)" link below the table.

4. Download all configuration files

To download the resulting configurations, you can use the "download all configurations (ZIP-Archive)" link located below the Template Value Set table at the Config Template view or the download link ( next to or below the Template Value Set name).

To show the configuration result, use the link.

5. Export the configurations to the local FTP/TFTP server

If this application is deployed using the integrated setup scripts, a FTP and TFTP server is configured by default. To export the configurations to the directories of this servers, simply go to "export configurations" in the Config Template view.

Please note: The password for the ftpuser is generated dynamically during the setup.

The sync with the local directories is always on-demand and can be triggered using the "refresh config" button.
